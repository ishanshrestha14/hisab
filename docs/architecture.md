## Package responsibilities

### `packages/shared`
Single source of truth for validation schemas and TypeScript types.
- Zod schemas: `createClientSchema`, `createInvoiceSchema`, `lineItemSchema`, etc.
- Inferred types: `CreateClientInput`, `CreateInvoiceInput`, etc.
- Imported in **both** `apps/api` (request validation) and `apps/web` (form types)

### `packages/db`
Prisma client and generated types.
- `prisma/schema.prisma` ‚Äî full database schema
- `src/index.ts` ‚Äî exports `prisma` singleton
- Scripts use `dotenv-cli` to load root `.env`

### `apps/api`
Hono REST API.
- Each route group is a separate `Hono` instance in `src/routes/`
- Mounted in `src/index.ts` with `app.route()`
- Auth guard applied per route group: `app.use("/api/clients/*", requireAuth)`
- Environment loaded via `--env-file=../../.env` in dev

### `apps/web`
React SPA.
- React Router v7 for routing
- `ProtectedRoute` wrapper checks Better Auth session before rendering
- All API calls go through `src/lib/api.ts` (handles cookies + JSON)
- Auth state via `useSession()` from Better Auth React client

## Database schema

```
User ‚îÄ‚îÄ< Session
     ‚îÄ‚îÄ< Account          (OAuth)
     ‚îÄ‚îÄ< Client ‚îÄ‚îÄ< Invoice ‚îÄ‚îÄ< LineItem
     ‚îÄ‚îÄ< Invoice
     ‚îÄ‚îÄ< ExchangeRate      (NPR rate cache)
```

Key design decisions:
- `Invoice.token` ‚Äî public cuid for portal links, never expose `Invoice.id`
- `ExchangeRate` ‚Äî cached per (base currency, date) to avoid hammering the exchange rate API
- `LineItem.total` ‚Äî stored (not computed) so historical totals survive rate/price changes

## Auth architecture

Better Auth handles sessions via **HTTP-only cookies** ‚Äî no JWT in localStorage.

- Sign up / sign in ‚Üí Better Auth sets a session cookie
- All API requests include `credentials: "include"` so cookie is sent automatically
- `requireAuth` middleware calls `auth.api.getSession()` and attaches `user` to Hono context
- Public portal routes (`/api/portal/:token`) skip `requireAuth` entirely

## Public portal

The client portal at `/portal/:token` is fully unauthenticated:
- `token` is a cuid generated by Prisma at invoice creation time
- The token is the only thing needed to view or mark an invoice as paid
- The internal `Invoice.id` is never exposed to the client
