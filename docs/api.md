# Hisab API Reference

Base URL: `http://localhost:3001` (dev) / `https://api.yourdomain.com` (prod)

All protected routes require an active session cookie (set automatically by Better Auth on login).

---

## Auth

Handled by Better Auth at `/api/auth/*`. Key endpoints:

| Method | Path | Description |
|--------|------|-------------|
| POST | `/api/auth/sign-up/email` | Register with email + password |
| POST | `/api/auth/sign-in/email` | Sign in with email + password |
| POST | `/api/auth/sign-out` | Sign out (clears session cookie) |
| GET | `/api/auth/session` | Get current session |
| GET | `/api/auth/sign-in/google` | Initiate Google OAuth |

## Clients

All routes require authentication.

### List clients
```
GET /api/clients
```
Returns all clients belonging to the authenticated user, ordered by creation date (newest first).

**Response**
```json
[
  {
    "id": "clx...",
    "name": "Jane Doe",
    "email": "jane@example.com",
    "company": "Acme Inc.",
    "country": "United States",
    "defaultCurrency": "USD",
    "createdAt": "2026-03-30T00:00:00.000Z"
  }
]
```

### Create client
```
POST /api/clients
```
**Body**
```json
{
  "name": "Jane Doe",
  "email": "jane@example.com",
  "company": "Acme Inc.",        // optional
  "country": "United States",    // optional
  "defaultCurrency": "USD"       // USD | GBP | EUR | NPR
}
```

### Update client
```
PUT /api/clients/:id
```
Same body as create (all fields optional). Returns 404 if client doesn't belong to user.

### Delete client
```
DELETE /api/clients/:id
```
Returns 404 if client doesn't belong to user.

## Invoices

All routes require authentication.

### List invoices
```
GET /api/invoices
```
Returns all invoices with client name and line items, newest first.

### Get invoice
```
GET /api/invoices/:id
```
Returns single invoice with full client and line items. 404 if not owned by user.

### Create invoice
```
POST /api/invoices
```
**Body**
```json
{
  "clientId": "clx...",
  "currency": "USD",
  "issueDate": "2026-03-30",
  "dueDate": "2026-04-30",
  "notes": "Payment via bank transfer",   // optional
  "lineItems": [
    {
      "description": "Web development — March",
      "quantity": 1,
      "unitPrice": 2000
    }
  ]
}
```
- Invoice number is auto-generated (`INV-001`, `INV-002`…)
- Public portal token is auto-generated by Prisma
- Line item `total` is computed as `quantity × unitPrice`

### Update invoice
```
PUT /api/invoices/:id
```
Same body as create minus `clientId` (all fields optional). Replaces all line items if provided.

### Update invoice status
```
PATCH /api/invoices/:id/status
```
**Body**
```json
{ "status": "PAID" }   // DRAFT | SENT | PAID | OVERDUE
```

### Delete invoice
```
DELETE /api/invoices/:id
```

## Public Portal

No authentication required. Token is the `Invoice.token` cuid.

### View invoice
```
GET /api/portal/:token
```
Returns invoice with client and line items. Safe for public — never exposes internal `id`.

### Mark as paid
```
POST /api/portal/:token/mark-paid
```
Sets invoice status to `PAID`. No body required.
